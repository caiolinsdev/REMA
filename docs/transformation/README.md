# Transformation

## Objetivo

Organizar a fase de transformation do REMA em `waves` pequenas, funcionais e
testaveis, seguindo a diretriz `Prompt -> Spec -> Code`.

## Waves

- `Wave 0`: [Foundation](./wave-0-foundation.md)
- `Wave 1`: [Auth and Shell](./wave-1-auth-and-shell.md)
- `Wave 2`: [Academic Core](./wave-2-academic-core.md)
- `Wave 3`: [Submission and Review](./wave-3-submission-and-review.md)
- `Wave 4`: [Content and Calendar](./wave-4-content-and-calendar.md)
- `Wave 5`: [Profile and Community](./wave-5-profile-and-community.md)
- `Wave 6`: [Games](./wave-6-games.md)

## Leitura Recomendada

1. `Wave 0`
2. `Wave 1`
3. `Wave 2`
4. `Wave 3`
5. `Wave 4`
6. `Wave 5`
7. `Wave 6`

## Regra de Execucao

Cada wave deve sair com:

- spec funcional
- spec tecnica
- contratos minimos
- criterio de pronto
- riscos principais
